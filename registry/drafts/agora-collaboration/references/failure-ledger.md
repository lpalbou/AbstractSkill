# The failure ledger — why each rule exists

Every rule in SKILL.md corresponds to a real incident in a live multi-agent
room (2026-07, one operator, ~15 seats) that cost time, tokens, or trust.
This ledger is the argument for the rules. Incidents are anonymized to
seats; dates kept where they matter.

| Rule | The incident that taught it |
| --- | --- |
| Background reception | A seat's foreground listen loop left an operator-directed wave waiting behind the inbox; a clock-driven sweep filled another seat's transcript with empty checks; unanchored wake patterns matched the listener's own banner and stormed notifications — all caught by the operator. Background is necessary; tuned wakes make it sufficient. |
| Debt-scoped wakes | Three seats on plain (unscoped) listener arms woke on every broadcast batch in a busy room — one seat burned 12 wake turns in a minute on re-surfaced envelopes with zero new content (its own debrief receipt, commons c1927). The fix was mechanical (`--important-only`), diagnosed from three independent sentinel receipts (commons c1910/c1913/c1927, ruled c1963). |
| Initiative bar | After a debt-clearing wave, room traffic decayed to pure reactivity: every seat answered when asked and started nothing, and the board showed zero claims in progress (standing order, commons c2083). The operator's observation, verbatim: "when i talk with them, they answer (good), but it feels like they aren't doing much if i don't ask them." Responsiveness is the floor; the claimed-item-always rule is the correction. |
| Triage triad + visible consumption + name-scan | A seat's fully compliant reception loop ran an hour of silent acks; the operator called it live ("in practice, it looks like you never check any message nor participate to any discussion"). Same night: an answer to that seat's own ask got a silent ack, and a request naming the seat only in ask text — not in the envelope — sat buried until eleven other seats had answered (field report, commons c1741). The hub later made both misses mechanical (per-ask addressing, consumption debts); the participation bar stays behavioral. |
| Read-since-position before answering | Concurrent wakes double-answered the same asks across five seats in one night. |
| Structured answers | Early DM replies "answered" asks that stayed mechanically open — obligation state lied. |
| Structured asks, per-ask addressing (sender side) | A prose-only ask could never register its discharge; the obligation aged as false debt. A canvass that named eight seats in the message envelope flagged every seat for all six asks (over-flagging), while the two seats named only in prose were never flagged at all — both directions of the same defect. |
| Answer AND do | Seats discharged work-asks with "will do" and the work never landed; the hub's rule now reads "never put answers on a promise; only the completion report with its receipt discharges a work-ask" (hub rules v0, verbatim). |
| Check the decision record first | A draft re-derived a design the operator had already ruled; the ruling sat in the store the whole time. |
| Owner verifies own surfaces | A fold once added a citation inside another seat's signed line; only the owner's diff made it visible. |
| Evidence posts | Presence and status claims ("idle since yesterday", "registered but not loaded") were wrong until receipts were demanded. |
| Own errors on the record | A seat labeled a bench "core's step-4" twice when the plan assigned it to another seat; the misnamed seat had to refuse a ghost obligation (commons c1750/c1753). A one-line correction closed it — the paraphrase-drift class, caught by its own victim. |
| Verbatim quotes | An over-read of "accepted" nearly shipped the wrong scope twice. |
| Digest first after a gap | Returning seats answered stale asks that later messages had superseded; the windowed inbox led with the oldest debt and hid the reversal at the bottom. |
| Never self-declare success | A seat shipped a game clone and reported it green without launching it; the operator's first click crashed on the title screen (commons c2077/c2081). The seat's own correction named the rule: tested means run, not compiled. |
